Clicking on a hyperlink to navigate seems simple, but in reality, it involves many steps, and the browser plays a crucial role in this process. Next, I will show you the underlying logic behind it.

  • 1. Parse the URL:

    The browser will parses the URL associated with the hyperlink to determine its destination(which is the attribute of href = " "), of course it includes resolving relative URLs to their absolute counterparts so that anchor links will work.

  • 2. Formulate an HTTP Request:

    The browser then formulates an HTTP GET (You can click on HTTP or GET to get more information) request for the resource. This request includes:

    Just think about an HTTP request as your browser connecting to the server and either asking for a specific resource or sending data to it. And there exist several http request types, like get,post,put,etc. But in the context of hyperlinks, get is the most common type; A GET request asks the server for a specific piece of information or resource. When you connect to a website, your browser usually sends several GET requests to receive the data that it needs for the page to load.

    HTTP is a protocol. In fact, the acronym stands for HyperText Transfer Protocol. This protocol governs the structure and language of the requests and responses that take place between clients and servers. The clients are usually web browsers, but they can come in many forms, such as search engine robots. When you visit websites through a browser, the entire connection takes place via HTTP. The protocol enables you to receive data, including text, images, videos, stylesheets, scripts, and more.

    1. The URL of the resource.
    2. Headers that provide context about the request, such as the user-agent (browser type), accepted languages,and more.
  • 3: Send the Request to the Server:

    Send the request to the server which contains target resuorces.

  • 4: Receive and Process the Server Response:

    The server responds with an HTTP response, which includes a status code (like 200 for a successful request) and the requested content. And then the browser process this response in different ways: Show that:

    1.If the content is an HTML page, the browser renders it.
    2. If it's a downloadable file, the browser may download it or ask the user how to proceed, based on the content-type header and browser settings.

  • 5: Render the Content:

    For HTML content, the browser parses the HTML and CSS, constructing the DOM (Document Object Model) and rendering the page for the user to view.
